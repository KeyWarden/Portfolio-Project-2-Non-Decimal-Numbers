Introduction

It is easy, at first, to fall into the trap of viewing non-decimal number systems, such as binary or hexadecimal, as strange or other in comparison to decimal numbers. After all, they have all these extra zeros in front of the numbers at times, and some even contain letters! But the truth is that all number systems, including decimal, run on the same logic, we're just taught decimal so completely, that reading it becomes second nature.

In this site, we hope to teach you not only how to read and understand two of the most prominent non-decimal numbers - binary with a base of 2, which is one of the most commonly known, and hexadecimal with a base of 16, which is used in many fields, such as in representing colours in CSS - we also hope to teach you the basics of reading and understanding any base number system.

How to read Non-Decimal Numbers

At the core, all number systems operate the same way. Each digit, from right to left, represents an ascending power of the base number, starting from a power of 0. Each digit can go up to 1 less than the base number. Finally, the end value of the number is calculated by multiplaying each digit by the base number to the power of its position, and then adding them together. Sounds tricky right?

Thankfully, things are simpler than they sound. Let's take a decimal number, 192, as an example. The three digits are powers of 10 in the following order: (power of 2), (power of 1), (power 0f 0). In other words, they represent multiples of 100, 10, and 1 respectively. Therefore, to get the final answer of one-hundred-and-ninety-two, we first take each of the given digits and multiply them by the position (1*100, 9*10, 2*1), giving 100, 90 and 2 respectively. Then we add them up, and get 192.

Therefore, if we were to instead take a base 16 system (hexadecimal), and use the same digits, 192, we can calculate an answer (1*16^2 + 9*16^1 + 2*16^0 = 1*256 + 9*16 + 2*1 = 256 + 144 + 2 = 402 in decimal).

However, we cannot use 192 in any base number system with a base lower than 10, because the highest any digit in a base number system can go is one less than the base number itself.

That covers the basics of how all number systems are framed, but where do the letters in some numbers, like hexadecimal, come from? As mentioned each digit can contain a value up to 1 less than the base, but when you exceed 10 in the base, this means needing to include a value we have no way to represent numerically except with multiple digits, which violates the underlying principle of all base number systems. Therefore, and alternative is needed, and provided: letters.

More specifically, letters A through Z can easily represent numbers 10 through 35 respectively with a single digit, so in number systems of base 11 through to 36, the relevant letters are used instead of double-digit numbers.

Things become more difficult past that point, so it is highly recommended you research any higher number system you may intend to use before commiting. They shall not be covered here regardless.

Additional Trivia

Beyond this, there is another category of base number system that we have not covered, and will not be testing you on, as it is more advanced: negative base numbers. Negative number systems are, at a glance, indistinguishable from positive ones. This is because you do not use negative numbers in the digits for a negative system, only positive ones that are still at most 1 less than the base would be if it were positive.

For example, in negabinary (base -2), you still only use 1s and 0s. However, each digit is a power of -2. To provide a further example: the number 3 in normal binary using 3 digits is 011 (0*4 + 1*2 + 1*1 = 2 + 1 = 3), but in negabinary it is 111 (1*(-2)^2 + 1*(-2)^1 + 1*(-2)^0 = 1*4 + 1(-2) + 1*1 = 4 - 2 + 1 = 3). This is interesting not only for the obvious reason of making the number inherently more complex, but also because it breaches a rule not touched on above.

In positive base numbers, a rule of thumb that always applies is that if the number is less than one of the values a digit represents, then the end result will require 1 fewer digits than that to convey. For example in decimal the third digit (right to left) represents 100, so if the number being represented is less than 100, it only needs to digits to represent, such as 99. In the prior binary example, because 3 is less than 4 and the third digit represents 4, only two digits are needed to convey it.

In negative systems, however, some of the digits represent negative values, meaning that you may require digits representing greater values to convey a smaller number due to the subtractions. In the given negabinary example, 3 requires 3 digits to convey because even though it is less than the value of the third digit - 4 - there is no way to add a -2 and 1 together in such a way as to create 3, meaning that you need 4 in order to make 3 possible.

Again, negative base numbers are not common and are very tricky to work with for obvious reasons, so we will not be testing you on them, but theya re truly fascinating to consider.
